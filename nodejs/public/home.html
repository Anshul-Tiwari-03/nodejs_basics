    # Wrapper Module

    Each and every file in node.js is wrapped in a function like :

    Suppose there is a command in index.js => console.log("hello world");

    function( exports , require , module , __filename, __dirname'){
    console.log('hello world');
    }

    This wrapping gives various features to node js

    exports : A reference to module.exports that can be used to make it accessible to others.
    require : A reference import Node js Modules.
    module : A reference to current module.
    __dirname : Directory name of the current file.
    __filename : File name of the current file.

    # FileSystem Module

    File System enables interacting with the files and filesystem based on POSIX function

    Promise Based API , Sync API , Callback Api

    const fs = require('fs') or require('fs/promises')

    Various Functions are

    access(path) : check if dir or file exists or not .
    mkdir(path) : creating directory
    readdir() : Read content of direcoty.
    remdir() : Remove directory.
    writeFile() : Writes file async if present then replaces it .
    appendFile() : Appends file async if not present then creates it .
    copyFile() : Copy File src to dest .
    stat() : give stats of the file .


    Code Example Directory Create Promise Based Api
    const fs = require('fs/promises');
    async function createDirectory(){
    const directoryPath = path.join(__dirname, 'demo');
    try {
      // Check if the directory already exists
      await fs.access(directoryPath);
      console.log('Directory already exists');
    } catch (error) {
      if (error.code === 'ENOENT') {
         // Directory doesn't exist, create it
         try {
            await fs.mkdir(directoryPath);
            console.log('Directory created');
          } catch (createError) {
            console.error('Error creating directory:', createError);
          }
      } else {
         console.error('Error accessing directory:', error);
      }
    }



    # OS Module

    It provides operating system-related utility and properties

    const os = require('os');

    Functions

    platform() : returns the string indentifying the operating system.
    arch() : returns the architecture for which linux was configured.
    cpus() : returns an array of objects containing information.
    hostname() : returns the host name of the operating system.
    freemem() : returns the amount of free memory in bytes as an integer.
    totalmem() : returns the amount of total memory in bytes as an integer.

    # EVENT Module

    It provides utilities for URL resolution and parsing

    const events= require('events');
    var eventEmitter= new events();

    Event emitter is registered using on() & once() method
    on() method
    eventEmitter.on('myevent',function(){
       console.log("An event just occurred");
    });
    eventEmitter.emit('myevent');
    eventEmitter.emit('myevent');
    This event will be triggered every time.
    once() method
    eventEmitter.once('myevent',function(){
       console.log("An event just occurred");
    });
    eventEmitter.emit('myevent');
    eventEmitter.emit('myevent');
    This event will be triggered one time.
